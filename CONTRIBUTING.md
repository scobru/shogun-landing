# Contributing to Shogun Landing

Thank you for your interest in contributing to the Shogun ecosystem! As a decentralized project, we welcome contributions from the community.

## Code of Conduct

Please be respectful and professional in all interactions. We strive to maintain a welcoming environment for everyone.

## How to Contribute

1.  **Fork the repository**: Click the "Fork" button at the top right of this page.
2.  **Clone your fork**: `git clone https://github.com/YOUR_USERNAME/shogun-landing.git`
3.  **Create a branch**: `git checkout -b feature/amazing-feature`
4.  **Make your changes**: Edit the code, add new project links, or fix bugs.
5.  **Test your changes**: Open `index.html` in your browser to verify the layout and links. Run `npm test` (or `npx html-validate index.html`) to ensure code quality.
6.  **Commit your changes**: `git commit -m 'Add Amazing Feature'`
7.  **Push to the branch**: `git push origin feature/amazing-feature`
8.  **Open a Pull Request**: Go to the original repository and click "New Pull Request".

## Reporting Bugs

If you find a bug, please open an issue using the Bug Report template. Include as much detail as possible:

-   Browser and OS version
-   Steps to reproduce
-   Screenshots (if applicable)

## Suggesting Enhancements

Have an idea for a new feature? Open an issue using the Feature Request template.

## Adding a New Project

To add a new project to the landing page:

1.  Add a new card in the appropriate section in `index.html`.
2.  Ensure you use the existing CSS classes for consistency (`minimal-card`, `minimal-link`, etc.).
3.  Verify that all links are correct and working.
4.  If available, add a link to the GitHub repository.
